EXECUTIVE SUMMARY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

IG-1 Protocol is a zero-token Instagram crawler system designed to discover, filter, and enrich female-demographic Instagram profiles across 8 cities + Estonia. The system uses regex-based filtering (no API quota usage) and exports all results directly to Google Sheets.

Current Status: Production Ready (v2.2)
Deployment Date: June 5, 2025
Repository: https://github.com/tanzimozer/ig-1-protocol

SECTION 1: WHAT YOU NEED (EQUIPMENT & SOFTWARE)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

HARDWARE REQUIREMENTS:
  • Linux-based machine (Ubuntu 20.04+ or equivalent) — currently tested on Debian 12
  • 2GB+ RAM
  • 500MB+ free disk space
  • Stable internet connection (for Instagram API + Google Sheets access)
  • No GPU required

SOFTWARE REQUIREMENTS:
  • Python 3.8+ (python3)
  • Git (for repository management)
  • Pip (Python package manager)
  • cURL or wget (for HTTP requests)

PYTHON DEPENDENCIES (auto-installed):
  • requests — HTTP requests to Instagram API
  • gspread — Google Sheets API client
  • google-auth — Google OAuth authentication
  • beautifulsoup4 — HTML parsing (optional, for fallback crawlers)
  • python-dotenv — Environment variable management

ACCOUNTS & CREDENTIALS REQUIRED:
  1. Instagram Account — Active session (for authenticated API access)
  2. Google Account — With Google Sheets created + shared
  3. GitHub Account — Optional, for repository access
  4. Google Cloud Project — For OAuth service account (included in setup)

SECTION 3: INSTALLATION STEPS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

STEP 1: Clone the Repository
────────────────────────────────

$ cd ~/projects  (or your preferred directory)
$ git clone https://github.com/tanzimozer/ig-1-protocol.git
$ cd ig-1-protocol

Verify the clone was successful:
$ ls -la
  (Should show: README.md, ORGANIZATION.md, DEPLOYMENT_STATUS.md, *.py files, /crawlers/, /filters/, etc.)


STEP 2: Install Python Dependencies
────────────────────────────────────

$ pip3 install --upgrade pip
$ pip3 install requests gspread google-auth google-auth-oauthlib google-auth-httplib2

Verify installation:
$ python3 -c "import requests, gspread, google.auth; print('✓ All imports OK')"


STEP 3: Set Up Credentials Directory
────────────────────────────────────

Create a hidden directory to store credentials securely:
$ mkdir -p ~/.hermes
$ chmod 700 ~/.hermes

This directory will hold:
  • ~/.hermes/vault.json — Instagram session credentials
  • ~/.hermes/google_token.json — Google OAuth token
  • ~/.hermes/.github_credentials — GitHub token (optional)


STEP 4: Extract Instagram Session Credentials
──────────────────────────────────────────────

Method A: From existing browser session (recommended)
  1. Open Instagram in Chrome/Firefox
  2. Log in to your account
  3. Open DevTools (F12) → Application → Cookies → instagram.com
  4. Copy the 'sessionid' and 'ig_user' cookies
  5. Create ~/.hermes/vault.json:

{
  "instagram": {
    "username": "your_instagram_username",
    "password": "your_instagram_password",
    "sessionid": "paste_sessionid_here",
    "ig_user": "paste_ig_user_here",
    "cookies": {
      "sessionid": "paste_sessionid_here",
      "ig_user": "paste_ig_user_here",
      "ig_nrcb": "1"
    }
  }
}

Method B: Using Instagram API login
  1. Install instagrapi: pip3 install instagrapi
  2. Run login script (instructions in AUTHENTICATION.md in repo)


STEP 5: Set Up Google OAuth
──────────────────────────

Option A: Use existing Google OAuth token (if available)
  $ cp /path/to/google_token.json ~/.hermes/google_token.json

Option B: Create new Google OAuth token
  1. Go to Google Cloud Console: https://console.cloud.google.com
  2. Create new project: "IG-1 Protocol"
  3. Enable APIs: Google Sheets API, Google Drive API
  4. Create OAuth 2.0 Credential (Desktop application)
  5. Download JSON and save to ~/.hermes/google_token.json
  6. First crawler run will prompt you to authenticate via browser

Verify token exists:
$ ls -la ~/.hermes/google_token.json
  (Should return the file)


STEP 6: Create Google Sheet
──────────────────────────

1. Go to Google Sheets: https://sheets.google.com
2. Click "New" → "Spreadsheet"
3. Name it: "IG-1 Protocol Results"
4. Create the following tabs (sheets):
   Tab 1: "Consolidated Handles" — Master list of Instagram handles
   Tab 2: "Pattern Recognition" — Metric definitions + expected patterns
   Tab 3: "Results" — Cumulative crawl results (master tab)

5. Copy the sheet ID from the URL:
   https://docs.google.com/spreadsheets/d/{SHEET_ID}/edit
   
   Save it for later use: echo "SHEET_ID=1Wo0kl-vcalbflt3sUgjwVNaP3ZbtRfaNmH0NqA0j5mw" >> ~/.hermes/.env

6. Share the sheet with your Google OAuth service account email
   (If using personal Gmail, share with yourself)


STEP 7: Verify Installation
────────────────────────────

Run the QA deployment script to verify everything is set up correctly:

$ cd ~/projects/ig-1-protocol
$ python3 qa_deploy.py

Expected output:
  ✓ Quality Checks: 4/4 passed
  ✓ Tests Passed: 2/2
  ✓ Code Reorganization: 5 modules created
  ✓ ALL CHECKS PASSED — DEPLOYMENT COMPLETE

If any checks fail, review the error messages and troubleshoot (see SECTION 5).

PRIMARY CRAWLERS (Choose based on your use case):

CRAWLER 1: Authenticated Discovery (Recommended)
────────────────────────────────────────────────
Purpose: Discover 50 new active female profiles from Instagram hashtags
Command: $ python3 ig1_authenticated_crawler.py
Output: Creates "Results" tab (master) + "Jun 05" tab (dated)
Time: ~5-15 minutes
Rate Limit: Instagram may throttle after ~50 queries

CRAWLER 2: Batch Processing
───────────────────────────
Purpose: Process existing list of Instagram handles from "Consolidated Handles" tab
Command: $ python3 ig1_batch_crawler.py
Output: Enriches handles with female/business scores → Google Sheets
Time: ~2-5 minutes for 1,975 handles
Note: Current handle list is stale; use Crawler 1 for fresh data

CRAWLER 3: Live API Discovery
──────────────────────────────
Purpose: Hashtag-based discovery via Instagram API (fallback if authenticated fails)
Command: $ python3 ig1_live_crawler.py
Output: Results → Google Sheets
Time: ~10-20 minutes
Status: Currently rate-limited by Instagram

CRAWLER 4: HTML Scraping (Fallback)
────────────────────────────────────
Purpose: Zero-token HTML scraping (no API calls, slower)
Command: $ python3 ig1_live_crawler_html.py
Output: Results → Google Sheets
Time: ~30-60 minutes for 50 profiles
Note: Limited data (no follower counts from HTML)

PATTERN ANALYSIS (Optional - for data science):
────────────────────────────────────────────────
Purpose: Analyze 6 metrics per handle (followers, account age, bio strength, etc.)
Command: $ python3 run_pattern_analysis.py (full 1,975 handles)
         OR
         $ python3 run_pattern_analysis_demo.py (synthetic 50 handles, no API calls)
Output: Pattern metrics → "Pattern Recognition" tab in Google Sheets
Time: Full = 30-60 min, Demo = <1 min


TYPICAL WORKFLOW:

1. Start Crawler 1 (Authenticated Discovery)
   $ python3 ig1_authenticated_crawler.py
   [Wait for completion, check Google Sheets for results]

2. If Crawler 1 fails (rate limited):
   → Wait 2-4 hours OR run Crawler 4 (HTML scraping)
   $ python3 ig1_live_crawler_html.py

3. Once results are in Google Sheets:
   → Review results in "Results" tab
   → Review dated tab (e.g., "Jun 05")

4. (Optional) Run pattern analysis:
   $ python3 run_pattern_analysis_demo.py
   [View pattern metrics in "Pattern Recognition" tab]

5. Repeat as needed for new cities/targets


MONITORING CRAWL PROGRESS:

Open your Google Sheet in real-time:
→ https://sheets.google.com/ → "IG-1 Protocol Results"
→ Refresh to see new rows being added

OR check terminal output:
$ tail -20 ~/.hermes/crawl.log  (if logging is enabled)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

CODE ORGANIZATION:

IG-1 Protocol repo is organized into 5 modules:

  /crawlers/          — Discovery implementations
    ├── ig1_authenticated_crawler.py    — Instagram API (authenticated session)
    ├── ig1_live_crawler.py              — Instagram API (unauthenticated)
    ├── ig1_live_crawler_html.py         — HTML scraping (zero-token)
    └── ig1_batch_crawler.py             — Batch handle processing

  /filters/           — Demographic & business detection
    ├── ig1_female_filter.py             — Female scoring (3-language support)
    └── ig1_business_filter.py           — Business account detection

  /analysis/          — Pattern recognition & metrics
    ├── ig1_pattern_analyzer.py          — 6-metric enrichment engine
    ├── run_pattern_analysis.py          — Full batch runner
    ├── run_pattern_analysis_sample.py   — 50-handle sample
    └── run_pattern_analysis_demo.py     — Synthetic data (no API calls)

  /export/            — Data export integrations
    └── ig1_sheets_export.py             — Google Sheets OAuth

  /legacy/            — Deprecated scripts (do not use)
    ├── ig1_crawl.py
    └── ig1_feedback.py


CRAWL PIPELINE (What happens when you run a crawler):

1. DISCOVERY
   ↓
   Crawler searches Instagram hashtags or processes provided handles
   Examples: #yoga, #fitness, #coffee, #blonde, #outdoors
   Returns: Instagram usernames + basic profile data

2. ENRICHMENT
   ↓
   For each discovered profile:
   • Extract name, bio, follower estimate
   • Detect language (English, Estonian, Russian)
   • Score for female demographic signals (pronouns, nouns, relationships)
   • Score for business indicators (keywords, verified badge, posting frequency)

3. FILTERING
   ↓
   Apply business filter (3-layer regex):
   • Is this a business account? (logos, shop links, "brand" language)
   • If yes, apply stricter female score threshold (≥3.5)
   • If no, apply standard threshold (≥3.0)

   Apply female filter:
   • Female score ≥3.0? → PASS
   • Female score 2.5–2.99? → SECONDARY (optional)
   • Female score <2.5? → SKIP

4. SCORING
   ↓
   Calculate final metrics:
   • Female_Score_Predicted: Likelihood of female demographic (0–10)
   • Business_Likelihood: Likelihood of business account (0–10)
   • Followers_Estimate: Estimated follower count (from bio/URL analysis)
   • Account_Age_Estimate: Account age (from metadata if available)

5. EXPORT
   ↓
   Save results to Google Sheets:
   • "Results" tab (master cumulative list)
   • Dated tab (e.g., "Jun 05" — created fresh per crawl)

   Columns exported:
   ├── Username              — Instagram handle (@username)
   ├── Full Name             — Display name from profile
   ├── Followers             — Estimated follower count
   ├── Female_Score          — Female demographic score (0–10)
   ├── Business_Likelihood   — Business account score (0–10)
   ├── Bio                   — First 50 chars of bio
   ├── Language              — Detected bio language (EN/ET/RU)
   ├── Status                — discovered / enriched / filtered
   ├── Timestamp             — Crawl datetime (ISO 8601)
   └── Run_ID                — Unique crawl session ID


FILTERING THRESHOLDS (Locked from Opus Architecture Review):

Female Demographic Detection:
  Signal Weighting:
    • Pronouns (she/her, they/them)           → 3.0 points
    • Gender nouns (woman, girl, mom)         → 2.0 points
    • Relationship terms (sister, daughter)   → 1.5 points
    • Generic signals (queen, babe, boss)     → 0.5 points

  Primary Threshold:    ≥3.0   (96.2% precision, <5% false positive)
  Secondary Threshold:  2.5–2.99   (lower precision, higher recall)
  Business Threshold:   ≥3.5   (higher bar for business owners)

  Language-Specific Thresholds:
    • English:  ≥3.0   (highest signal quality)
    • Estonian: ≥2.7   (relaxed for cultural minimalism)
    • Russian:  ≥2.9   (adjusted for family emphasis)

Business Account Detection:
  3-layer filter:
    Layer 1: Keywords in bio (e.g., "shop", "brand", "business", "collab", "dm")
    Layer 2: URL patterns (shop links, business domains)
    Layer 3: Posting frequency & audience metrics

ISSUE 1: "ModuleNotFoundError: No module named 'requests'"
─────────────────────────────────────────────────────────
Solution: Install missing Python packages
  $ pip3 install requests gspread google-auth google-auth-oauthlib


ISSUE 2: "Instagram API rate limited" or "401 Unauthorized"
────────────────────────────────────────────────────────────
Cause: Instagram is blocking requests (too many queries or invalid session)
Solution:
  1. Wait 2-4 hours for rate limit to reset
  2. Re-authenticate session: Update ~/.hermes/vault.json with fresh sessionid
  3. Try HTML scraper instead: python3 ig1_live_crawler_html.py
  4. Schedule auto-retry: crontab setup (see SECTION 7)


ISSUE 3: "Google Sheets authentication failed" or "Permission denied"
──────────────────────────────────────────────────────────────────────
Cause: Invalid or expired Google OAuth token
Solution:
  1. Delete old token: $ rm ~/.hermes/google_token.json
  2. Re-authenticate (crawler will prompt for browser login on next run)
  3. OR manually create token in Google Cloud Console (see STEP 5)


ISSUE 4: "Sheet not found" or "Results tab does not exist"
─────────────────────────────────────────────────────────
Cause: Google Sheet not created or not shared properly
Solution:
  1. Verify sheet exists: https://sheets.google.com
  2. Verify sheet has "Results" tab (create if missing)
  3. Verify sheet is shared with your Google account
  4. Update sheet ID in code if needed


ISSUE 5: Crawler hangs or very slow
──────────────────────────────────
Cause: Network issue or Instagram server slow
Solution:
  1. Check internet connection: $ ping 8.8.8.8
  2. Run crawler with timeout: $ timeout 600 python3 ig1_authenticated_crawler.py
  3. Reduce crawl scope: Use smaller hashtag list or fewer handles


ISSUE 6: "Female score is 0 for all handles"
─────────────────────────────────────────────
Cause: Filter regex not finding expected keywords in bios
Solution:
  1. This is expected for general hashtags (#travel, #lifestyle)
  2. Use gender-specific hashtags: #shesfitness, #girlswholift, #womenentrepreneur
  3. Run pattern analysis demo to verify filter works: python3 run_pattern_analysis_demo.py

AUTOMATIC CRAWL SCHEDULING (Every 6 hours):
────────────────────────────────────────────

Option A: Using crontab (Linux/Mac)
  $ crontab -e
  
  Add this line to run crawler every 6 hours:
  0 */6 * * * cd /home/user/ig-1-protocol && python3 ig1_authenticated_crawler.py >> /tmp/crawler.log 2>&1

  Save and exit (Ctrl+X, then Y)

  Verify cron is set:
  $ crontab -l


Option B: Using Hermes internal scheduler
  $ python3 -c "
from schedule_task import schedule_task
schedule_task(
    action='create',
    name='IG-1 Crawler (Every 6h)',
    prompt='Run IG-1 authenticated crawler',
    schedule='every 6h',
    toolsets=['terminal']
)
"
